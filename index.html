Getting started
ติดตั้ง Cariva DS

คัดลอก prompt นี้แล้ววางใน Claude Code — AI จะถามและดำเนินการให้ทั้งหมด ตั้งแต่เลือก path ไปจนถึงเปิดพร้อมใช้งาน

Claude Code — install prompt 
ช่วยติดตั้ง Cariva Design System จาก repo นี้ให้หน่อย:
https://github.com/PraewNAT/cariva-ds

โดย:
1. ถามฉันก่อนว่าอยากติดตั้งไว้ที่ path ไหน
2. clone repo ไปไว้ที่ path นั้น
3. รัน npm install
4. เปิด Claude Code ในโฟลเดอร์นั้น
5. อ่าน CLAUDE.md และบอกฉันว่าต้องทำอะไรต่อ
💡 หลังติดตั้งแล้ว เปิด Claude Code ในโฟลเดอร์ cariva-ds — ระบบจะถาม role ให้เลือกอัตโนมัติทุก session
Role system
เลือก role ก่อนเริ่มงาน

เมื่อเปิด Claude Code ในโฟลเดอร์ cariva-ds ระบบจะถาม role ให้เลือกเสมอ — ทั้งสอง role ใช้ skill ได้เหมือนกันทั้งหมด ต่างกันแค่สิทธิ์แก้ไขไฟล์

Role 1

Design System Owner

ใช้ skill ได้ทั้งหมดและแก้ไขไฟล์ได้ทุกอย่าง

  • แก้ tokens.json, theme.ts, fonts.ts
  • สร้างและแก้ไข Core component
  • อัปเดต CLAUDE.md และ permission
  • push ขึ้น GitHub ได้
Role 2

UX/UI Designer

ใช้ skill ได้ทั้งหมด แต่อ่านไฟล์อย่างเดียว

  • รัน skill ทุกอันได้เต็มที่
  • ส่งไฟล์ .md ให้ Owner review
  • ไม่สามารถแก้ไข/สร้าง/ลบไฟล์
  • git pull cariva-ds ก่อนเริ่มงานทุกวัน
🔒 UX/UI Designer ห้ามขอแก้ไขไฟล์หรือเปลี่ยน permission เอง — ถ้าสั่ง AI จะปฏิเสธและแจ้งให้ติดต่อ Design System Owner

FAQ

คำถามที่เจอบ่อย

Figma ส่ง token binding มาเฉพาะตอนที่ส่ง node-id ของ component เดี่ยวๆ เท่านั้น ถ้าส่ง section หรือ page จะได้แค่โครงสร้าง — ให้ copy node-id ของ component set โดยตรงแทน AI จะแจ้งและขอ node-id ใหม่อัตโนมัติ
design-reviewhandoff
ใช้เมื่อยังอยู่ในขั้นตอน QAพร้อมส่งงานแล้ว
outputรายการปัญหา + ข้อเสนอแนะDev spec + QA checklist แยก 2 ชุด
ส่งให้ใครdesigner / PMdeveloper + QA
design-review, handoff, ux-writing, changelog, mapping — ใช้ได้กับทุก project ที่ไม่ได้ใช้ Cariva DS โดยไม่ต้อง config เพิ่ม
  1. สร้าง component ใน Figma ตาม convention crvComponent/variantName
  2. รัน audit เพื่อตรวจ token binding และ naming
  3. รัน component-doc เพื่อสร้าง doc และไฟล์ .md
  4. ส่งไฟล์ .md ให้ Design System Owner — Owner จะ review และ merge เข้า repo

หมายเหตุ: UX/UI Designer ไม่สามารถ push หรือแก้ไขไฟล์ใน repo ได้โดยตรง
เปิด Claude Code session ใหม่ — AI จะอ่าน tokens.json ล่าสุดอัตโนมัติ ไม่ต้องทำอะไรเพิ่ม
